# Kibana

**Автор:** Oleborn

**Дата:** 01 марта 2026 г.

**Проект:** [OrderHub](https://github.com/Oleborn/OrderHub)

Kibana — это мощный инструмент визуализации и анализа данных с открытым исходным кодом, который является неотъемлемой частью Elastic Stack. Она позволяет пользователям исследовать данные Elasticsearch, создавать интерактивные дашборды, мониторить производительность приложений, управлять кластером и многое другое. В контексте Observability, Kibana является центральным узлом, где инженеры и аналитики могут получить полное представление о состоянии своих систем, коррелируя логи, метрики и трассировки.

## 1. Фундаментальные концепции и архитектура

Kibana — это клиент-серверное приложение, которое взаимодействует с Elasticsearch для получения и отображения данных.

### 1.1. Архитектура Kibana

*   **Kibana Server (Backend)**: Написан на Node.js. Отвечает за:
    *   Обработку HTTP-запросов от браузера.
    *   Проксирование запросов к Elasticsearch.
    *   Управление сессиями и аутентификацией.
    *   Хранение конфигураций дашбордов, визуализаций и других объектов в индексе `.kibana` в Elasticsearch.
    *   Выполнение серверной логики для некоторых плагинов (например, Alerting, Reporting).

*   **Kibana Client (Frontend)**: Веб-приложение, написанное на React/Angular (в зависимости от версии) и использующее Elastic UI Framework. Отвечает за:
    *   Рендеринг пользовательского интерфейса.
    *   Отправку запросов к Kibana Server.
    *   Интерактивное взаимодействие с пользователем (фильтрация, сортировка, drill-downs).

*   **Индекс `.kibana`**: Специальный индекс в Elasticsearch, где Kibana хранит все свои объекты: дашборды, визуализации, сохраненные поиски, индексные паттерны, конфигурации и т.д. Это позволяет легко бэкапить и восстанавливать конфигурацию Kibana вместе с данными Elasticsearch.

### 1.2. Роль Kibana в Observability

Kibana объединяет три столпа Observability:

*   **Логи**: Централизованный просмотр, поиск и анализ структурированных логов из Elasticsearch. Корреляция логов по `traceId` и `spanId`.
*   **Метрики**: Визуализация метрик производительности систем и приложений, собранных Prometheus/Micrometer и хранящихся в Elasticsearch.
*   **Трассировки**: Интеграция с Jaeger/OpenTelemetry для визуализации распределенных трассировок, позволяющая находить узкие места в микросервисных архитектурах.

## 2. Подключение к данным

Прежде чем визуализировать данные, Kibana должна знать, откуда их брать и как интерпретировать.

### 2.1. Индексные паттерны (Index Patterns)

Индексный паттерн — это шаблон, который Kibana использует для идентификации одного или нескольких индексов Elasticsearch, которые вы хотите исследовать. Он позволяет работать с группами индексов как с единым целым (например, `logs-*`, `metrics-*`, `traces-*`).

*   **Создание**: При создании индексного паттерна вы указываете поле, содержащее временную метку (например, `@timestamp`), которое Kibana будет использовать для фильтрации данных по времени.
*   **Управление**: Индексные паттерны можно создавать, редактировать и удалять через раздел Management -> Index Patterns.

### 2.2. Источники данных (Data Sources)

Хотя Kibana по умолчанию работает с Elasticsearch, она также может подключаться к другим источникам данных через плагины (например, SQL-базы данных, Prometheus, Splunk, InfluxDB). Это расширяет возможности визуализации и позволяет создавать дашборды, объединяющие данные из разных систем.

## 3. Discover: исследование и поиск данных

Раздел Discover — это отправная точка для исследования данных в Elasticsearch. Он предоставляет мощные инструменты для поиска, фильтрации и анализа сырых документов.

### 3.1. Kibana Query Language (KQL)

KQL — это простой, но мощный язык запросов, разработанный специально для Kibana. Он позволяет легко фильтровать данные по полям, используя автодополнение и логические операторы.

*   **Базовый синтаксис**: `field:value` (например, `level:ERROR`)
*   **Поиск по тексту**: `message:"database connection error"`
*   **Логические операторы**: `AND`, `OR`, `NOT` (например, `level:ERROR and service.name:order-service`)
*   **Диапазоны**: `response_time_ms > 1000`
*   **Wildcards**: `service.name:order*`

### 3.2. Фильтрация и сортировка

*   **Time Range Selector**: Позволяет выбрать временной диапазон для отображаемых данных (например, "Last 15 minutes", "Today", "Custom range").
*   **Filters**: Можно создавать фильтры на основе значений полей (например, `level is ERROR`). Фильтры могут быть включены, выключены или инвертированы.
*   **Saved Searches**: Сохранение часто используемых запросов и фильтров для быстрого доступа.
*   **Сортировка**: Сортировка документов по любому полю (по умолчанию по `@timestamp`).

### 3.3. Корреляция логов с трассировками

В контексте Observability, ключевой функцией Discover является возможность корреляции логов с трассировками. Если ваши логи содержат `traceId` (как в проекте OrderHub), вы можете:

1.  Найти интересующий лог (например, ошибку).
2.  Скопировать `traceId` из этого лога.
3.  Использовать `traceId` для поиска соответствующей трассировки в Jaeger (или в Kibana APM, если настроено).

Это позволяет мгновенно переходить от симптома (ошибка в логе) к причине (полный путь запроса в трассировке).

## 4. Visualize

Раздел Visualize позволяет создавать различные типы графиков и диаграмм для представления данных.

### 4.1. Типы визуализаций

Kibana предлагает широкий спектр встроенных визуализаций:

*   **Гистограммы (Histogram)**: Для отображения распределения числовых данных или данных по времени (например, количество логов в минуту).
*   **Линейные графики (Line Chart)**: Для отображения временных рядов (например, QPS, Latency).
*   **Круговые диаграммы (Pie Chart)**: Для отображения долей (например, распределение ошибок по сервисам).
*   **Таблицы (Data Table)**: Для отображения агрегированных данных в табличном виде.
*   **Метрики (Metric)**: Для отображения одного числового значения (например, текущий QPS).
*   **Облака тегов (Tag Cloud)**: Для визуализации частоты встречаемости терминов.
*   **Карты (Maps)**: Для отображения геопространственных данных (требует GeoIP).
*   **Heat Map**: Для отображения плотности данных.

### 4.2. Агрегации и метрики

Каждая визуализация строится на основе агрегаций Elasticsearch. Вы выбираете:

*   **Bucket Aggregations**: Группируют документы по категориям (например, `Date Histogram` по `@timestamp`, `Terms` по `service.name.keyword`).
*   **Metric Aggregations**: Вычисляют значения по группам (например, `Count`, `Average`, `Sum`, `Max`, `Min`, `Percentiles`).

### 4.3. Продвинутые визуализации

Для создания кастомных и сложных визуализаций Kibana предоставляет мощные инструменты:

*   **Vega / Vega-Lite**: Позволяет создавать любые типы графиков с помощью декларативного языка JSON. Вы можете полностью контролировать внешний вид и интерактивность визуализации, используя данные из Elasticsearch. Это идеальный инструмент для уникальных бизнес-дашбордов, которые не вписываются в стандартные шаблоны Kibana.
    *   **Пример использования**: Создание кастомной диаграммы Ганта для отображения длительности спанов в трассировках, или сложной сетевой диаграммы для визуализации зависимостей сервисов.

*   **Canvas**: Инструмент для создания динамических, пиксель-совершенных презентаций и инфографики. Canvas позволяет комбинировать данные из Elasticsearch с изображениями, текстом, SVG-графикой и другими элементами, создавая интерактивные отчеты. Идеально подходит для Executive Dashboards или NOC-экранов.
    *   **Пример использования**: Создание "здоровья" сервиса, где каждый микросервис представлен карточкой с текущими метриками (QPS, Latency, Error Rate) и цветовой индикацией состояния.

## 5. Dashboards

Дашборды — это коллекции визуализаций, которые предоставляют комплексное представление о ваших данных. Они позволяют быстро отслеживать ключевые метрики, выявлять тенденции и реагировать на проблемы.

### 5.1. Создание и организация дашбордов

*   **Добавление визуализаций**: Просто перетаскивайте сохраненные визуализации на дашборд.
*   **Изменение размера и расположения**: Настраивайте макет дашборда для оптимального отображения информации.
*   **Фильтрация**: Дашборды наследуют фильтры по времени и KQL-запросы из Discover, а также позволяют добавлять свои собственные фильтры.

### 5.2. Динамические дашборды с переменными (Variables)

Переменные позволяют создавать интерактивные дашборды, где пользователи могут выбирать значения из выпадающих списков (например, `service.name`, `environment`) для динамической фильтрации данных.

*   **Типы переменных**: `Term`, `Query`, `Custom`.
*   **Использование**: Переменные используются в запросах визуализаций (например, `service.name:$service_name`).

### 5.3. Drill-downs и интерактивность

Kibana позволяет настраивать drill-downs, чтобы пользователи могли переходить от одной визуализации к другой, или даже к внешним URL, передавая контекст (например, `traceId`).

*   **URL Drill-downs**: Переход из дашборда Kibana в Jaeger UI, передавая `traceId` для просмотра полной трассировки.

## 6. Kibana Lens

Kibana Lens — это интуитивный, drag-and-drop редактор визуализаций, который упрощает процесс создания графиков, особенно для новичков. Он автоматически предлагает наиболее подходящие типы визуализаций на основе выбранных полей.

*   **Простота использования**: Перетаскивайте поля в области `X-axis`, `Y-axis`, `Break down by` и Lens автоматически построит график.
*   **Быстрые инсайты**: Позволяет быстро экспериментировать с данными и находить закономерности без глубокого знания агрегаций Elasticsearch.

## 7. Grafana Alerting (Unified Alerting)

Хотя Kibana имеет свои собственные возможности оповещения, современный подход в Elastic Stack — это **Unified Alerting**, который позволяет создавать, управлять и маршрутизировать оповещения из различных источников (метрики, логи, трассировки) в одном месте.

*   **Alert Rules**: Определяют условия, при которых должно сработать оповещение (например, `количество ошибок > N за M минут`).
*   **Connectors**: Определяют, куда отправлять оповещения (Slack, PagerDuty, Email, Webhook).
*   **Action Groups**: Группировка коннекторов для отправки оповещений разным командам.

## 8. Безопасность и пространства (Spaces)

Kibana предоставляет мощные функции безопасности для управления доступом к данным и объектам.

### 8.1. Управление доступом на основе ролей (RBAC)

*   **Пользователи и Роли**: Создание пользователей и назначение им ролей, которые определяют, к каким индексам и функциям Kibana они имеют доступ.
*   **Гранулярный контроль**: Возможность контролировать доступ на уровне индексов, документов и даже полей.

### 8.2. Пространства (Spaces)

Spaces позволяют создавать изолированные рабочие области в Kibana. Каждый Space имеет свои собственные дашборды, визуализации, сохраненные поиски и индексные паттерны. Это идеально подходит для больших организаций, где разные команды работают с разными наборами данных или требуют разных представлений.

*   **Изоляция**: Объекты, созданные в одном Space, не видны в другом.
*   **Контроль доступа**: Можно назначать разные роли пользователям для разных Spaces.

## 9. Dev Tools

Kibana включает встроенные инструменты для разработчиков, которые упрощают взаимодействие с Elasticsearch и отладку конфигураций.

*   **Console**: Позволяет отправлять HTTP-запросы к Elasticsearch напрямую из браузера. Идеально подходит для тестирования Query DSL, управления индексами и выполнения административных задач.
*   **Grok Debugger**: Инструмент для тестирования Grok-паттернов, который помогает парсить неструктурированные логи и создавать правильные регулярные выражения для Logstash.

## 10. Управление и провижининг (Provisioning)

В production-средах ручное управление Kibana становится неэффективным. Provisioning позволяет автоматизировать настройку Kibana.

### 10.1. Dashboard as Code (IaC для Kibana)

*   **Экспорт/Импорт**: Объекты Kibana (дашборды, визуализации) можно экспортировать в JSON-файлы и импортировать обратно. Это позволяет версионировать их в Git.
*   **Kibana API**: Kibana предоставляет REST API для программного управления всеми своими объектами. Это позволяет автоматизировать создание и обновление дашбордов.
*   **`grafana-sculptor` / `jsonnet` (Grafonnet)**: Хотя эти инструменты чаще ассоциируются с Grafana, их принципы (генерация JSON-конфигураций кодом) применимы и к Kibana. Можно использовать скрипты для генерации JSON-файлов дашбордов Kibana.

### 10.2. ILM (Index Lifecycle Management) и SLM (Snapshot Lifecycle Management)

Kibana предоставляет интерфейс для управления ILM (описанным в Elasticsearch Deep Dive) и SLM (автоматическое создание и удаление снапшотов кластера Elasticsearch). Это критически важно для управления жизненным циклом данных и обеспечения их сохранности.

## 11. Производительность и оптимизация

Оптимизация Kibana важна для обеспечения быстрого и отзывчивого пользовательского интерфейса.

*   **Оптимизация запросов Elasticsearch**: Самый большой фактор производительности. Убедитесь, что запросы к Elasticsearch эффективны (используют `keyword` поля для фильтрации, агрегации по `Doc Values`).
*   **Размер временного диапазона**: Чем меньше временной диапазон, тем быстрее запросы.
*   **Количество визуализаций на дашборде**: Слишком много визуализаций могут замедлить рендеринг. Используйте Lazy Loading для больших дашбордов.
*   **Кэширование**: Kibana кэширует результаты запросов. Оптимизируйте настройки кэша.
*   **Серверные ресурсы**: Убедитесь, что Kibana Server имеет достаточно CPU и RAM.
*   **Настройки браузера**: Используйте современные браузеры, очищайте кэш.

## 12. Интеграция с экосистемой Elastic Stack

Kibana является центральным хабом для всего Elastic Stack, предоставляя специализированные приложения для различных задач Observability.

*   **APM (Application Performance Monitoring)**: Мониторинг производительности приложений, трассировки, метрики и ошибки.
*   **SIEM (Security Information and Event Management)**: Анализ событий безопасности, обнаружение угроз.
*   **Logs**: Специализированное приложение для просмотра и анализа логов.
*   **Metrics**: Специализированное приложение для просмотра и анализа метрик.
*   **Uptime**: Мониторинг доступности сервисов.
*   **Maps**: Визуализация геопространственных данных.
